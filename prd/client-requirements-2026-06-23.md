# 客户需求结构化提取（2026-06-23）

> [!info] 文档定位
> 本文件是**独立需求提取**，基于客户发来的 HTML 看板原型（2026-06-17 版），与现有 `prd/PRD.md`（13F 交叉持股交付物）做差异对比。
> 原型明确标注「⚠ 当前为示例数据，仅用于验证看板结构与交互，非真实财务数据」——故本提取以**结构 / 字段 / 交互**为主，数据值仅作占位参考。

## §1 看板整体结构

### 1.1 顶部栏（topbar）

- **主标题**：竞品资本市场情报看板
- **副标题**：`IR 部门 · 数据源：SEC / HKEXnews / 官方披露 + 媒体 / 公众号（公开）`
  - 即数据源被显式声明为三大类：① SEC ② HKEXnews ③ 官方披露 + 媒体 / 公众号
- **两个标签页（tab，切换关系，非并列）**：
  - ① 单竞品档案（`tab-single`）
  - ② 行业横向对比（`tab-compare`）
- **全局公告条（notice）**：黄底，固定提示「当前为示例数据…非真实财务数据」。交付时需替换为真实数据时效/免责说明。
- **品牌色系**：深绿 `#006B3E` / 绿 `#00AC00` / 浅绿底 `#E8F5E9`；正负值用绿/红；行业均值用蓝 `#0000FF`；瓜子对标主体全局加 ★ 标记并高亮。

### 1.3 视觉/交互约定

- **未上市竞品**：仅渲染「当季跟踪」+ `na` 占位块，明确写「无强制披露义务…不臆测、不补值」。
- **瓜子对标主体**：全程 ★ 高亮，未上市但在横向对比矩阵中作为独立列出现，估值列显示「未上市」占位。
- **图表**：原型用纯 SVG 自绘（折线 / 柱状 / 散点），柱状图带「行业均值」虚线参考线；散点图为「营收增速 YoY% × PS(TTM)」。
- **YoY 着色**：同比为正绿、为负红；比率类用 pp、倍数类用 x、金额类用 %。

## §2 Tab ① 单竞品档案 — 详细字段清单

渲染入口 `renderSingle(id)`。竞品选择 chips → 档案头 → 快照条 → 要点 → 当季跟踪 → 5 个编号区块。

### 2.0 档案头（chead）

| 字段 | 来源 | 说明 |
|------|------|------|
| 公司名称 `nm` | config | 瓜子加 ★、深绿色 |
| 交易所 `exch` | config | 如 NYSE / HKEX / NASDAQ / NYSE·HKEX / 未上市 |
| Ticker `ticker` | config | 如 CVNA / ATHM·2518 / LI·2015 / 2858 |
| 财务口径 `fx` | config | 如「美元，折 RMB（principal 模式）」「港币，财务口径 RMB」「NEV 整车厂；口径 RMB」 |
| 上市徽标 `badge` | config | 已上市 / 未上市 / 对标主体（未上市） |

### 2.1 快照条（snapshot）— 仅上市竞品渲染

| stat 标签 | 单位 | 含义 | 数据源 | 对应现有模块 |
|-----------|------|------|--------|-------------|
| 最新股价 | 元/美元 | 期末股价 + YoY% | 免费延迟行情 | **新增**（collector 无行情） |
| 市值（亿元） | 亿元 | 期末市值 | 行情 × 股本 | **新增** |
| PS(TTM) | x | 市值 / TTM 营收 | 测算 | **新增** |
| PE(TTM) | x | 市值 / TTM 净利（亏损显示 N/A） | 测算 | **新增** |
| 营收（季,亿元） | 亿元 | 当季营收 | 财报 | collector.py ✅ |
| 净利率 | % | 当季净利率（盈利/亏损标注） | 测算 | collector.py ✅ |
| GMV（季,亿元） | 亿元 | 当季 GMV | 财报/公告 | **部分新增** |

### 2.2 本季要点（takeaways）

自动生成的 4 条要点：营收(亿元)+同比 / 净利润+净利率+环比 / GMV+同比 / PS(TTM) vs 行业均值。
未上市时第 4 条替换为「未上市 → 无市场估值、机构持仓与卖方覆盖（相关模块占位）」。

### 2.3 当季跟踪 · 26Q2（进行中，财报未发布）— intelSection

**这是新增的核心模块**：在财报未发布的当季，按可信度三层组织情报。

| 层级 | 名称 | 信息源 | 可信度 | 示例 |
|------|------|--------|--------|------|
| tier 1 | ① 官方披露层 | SEC / 港交所 / 公司公告 / IR | 最高 | 8-K/6-K 公告、IR 月度交付量、回购/可转债/ABS 发行 |
| tier 2 | ② 权威信息层 | 权威财经媒体 / 卖方研报 / 交易所互动 | 中 | 路透/彭博报道、券商上调目标价 |
| tier 3 | ③ 市场观察层 | 新闻 / 公众号 / 社媒 / 行业传闻 | 参考·待证实 | 雪球散户讨论、公众号「传…（待证实）」 |

每条情报字段：`tier` / `date` / `src`（来源）/ `cat`（资本·业务·财务 三类标签）/ `text`。
**对未上市竞品（瓜子/汽车街/大搜车/淘车车）同样渲染此模块**，是其主要信息来源。

### 2.4 区块 1 · 业务数据

- **KPI 卡片**：GMV(亿元) + YoY / 交易量(万台) + YoY / 变现率(%) + YoY(pp) / 单车收入(元/台) + YoY
- **年度+三年逐季明细表（ayTable）**：GMV / 交易量 / 变现率 / 单车收入，每格含 YoY（pp / %）
- **趋势图**：GMV 趋势折线 + 单车收入趋势折线（两列）
- 数据源标注：「来源：公司财报 / 官方披露」
- **对应现有**：collector.py 只覆盖 SEC 标准财务；**GMV/交易量/变现率/单车收入属业务指标，美股非强制 XBRL，需新增提取（8-K/press release/6-K/公众号）**

### 2.5 区块 2 · 财务数据

- **KPI 卡片**：营收 + YoY / 毛利率 + YoY(pp) / 净利润(盈利/亏损) / 净利率 + YoY(pp) / 经营现金流 + YoY / 现金 / 有息负债
- **ayTable**：营收 / 毛利率 / 净利润 / 净利率 / 经营现金流 / 现金 / 有息负债
- **趋势图**：营收+净利润双线 / 毛利率+净利率双线
- 数据源标注：「来源：SEC EDGAR / HKEXnews」
- **对应现有**：collector.py ✅（美股）；港股需新增 HKEXnews 抓取 + IFRS/HKFRS 口径转 RMB

### 2.6 区块 3 · 资本市场 · 估值

- **KPI 卡片**：市值 + YoY / 股价 + YoY / PE(TTM) / PS(TTM) / EV/Sales
- **ayTable**：股价(年末) / 市值 / PE / PS / EV/Sales（测算，含同比）
- **趋势图**：市值趋势 / PS(TTM) 趋势
- **重大资本动作（timeline）**：日期 + 事件（回购/股息/再融资/可转债/ABS 等）
- 注释：「股价为免费延迟行情；倍数按公开数据测算。年度 PE/PS 以年末市值÷全年净利/营收；前瞻倍数依赖付费一致预期，本版暂缺。」
- 未上市 → `na` 占位
- **对应现有**：行情/估值/资本动作**全部新增**；资本动作可从 8-K/6-K 提取

### 2.7 区块 4 · 投资人结构（前 50 大 · 按性质）— holderStructure

**13F 在新模版中的新语义**：不再是跨竞品交叉持股矩阵，而是**单竞品视角**的前 50 大持有人 + 性质构成。

- **持仓性质构成横条（hcomp）**：8 类色块横条，每行 = 色块 + 标签 + 占比% + 家数 + 环比 pp
- **8 类投资人性质（HCAT，替换现有 3 类 Index/Active/Broker）**：

| key | 标签 | 色 |
|-----|------|----|
| `insider` | 创始团队/内部人 | 深绿 |
| `strategic` | 战略/产业资本 | 蓝 |
| `longonly` | 主动多头(长线) | 橙 |
| `passive` | 被动指数 | 灰蓝 |
| `hedge` | 对冲基金 | 红 |
| `sovereign` | 主权/养老金 | 紫 |
| `insurance` | 险资 | 青绿 |
| `qfii` | QFII/外资资管 | 金 |

- **前 50 大持有人表（stick）**：排名 / 持有人 / 性质 / 持股% / 较上季 pp；含「前 50 大合计」行 + 「其余自由流通 ≈ X%」
- **集中度指标**：前五 / 前十 / 前二十 / 前五十 合计占比
- **持仓结构与资本市场动态分析（takeaways，三段）**：
  - 一、结构特征与成因（锁定盘/被动 vs 主动/对冲占比/机构覆盖度 → 自由流通盘、定价效率判断）
  - 二、近期变化与解读（环比变动最大类别、长线/对冲/被动资金流向解读）
  - 三、对未来股价波动的影响及原因（波动预判：偏高/中等/中等偏低；上行情形 + 下行风险）
- 数据源标注：「13F / 13D-G · 季度披露，滞后约 45 天」
- **对应现有**：`institutional_tracker.py` + `cross_holding.py` 部分可复用（13F 解析、QoQ），但需 ① 新增 8 类性质映射 ② 聚合到单竞品前 50 ③ 港股持有人需 CCASS 数据（非 13F）

### 2.8 区块 5 · 投资人主流观点

- **综合评级**（如 买入/中性/分歧大/中性偏多）
- **评级分布**（买入 X / 中性 Y / 卖出 Z）
- **目标价（公开引用）**
- **近期评级动作（timeline）**：日期 + 机构 + 评级/目标价变动
- **看多逻辑**（bull 列表）+ **看空/风险**（bear 列表）
- 注释：「为公开新闻聚合的零散观点，非完整 consensus。」
- **对应现有**：**全新模块** — 卖方研报/评级聚合，无现成数据源

### 2.9 未上市竞品渲染规则（noData / preIPO）

- 渲染档案头 + 当季跟踪（§2.3）+ `na` 占位
- 占位文案：「该竞品 未上市/IPO 前，无强制披露义务。公开信息：{businessNote}。→ 以上"当季跟踪"为主要信息来源，其余占位，不臆测、不补值。」
- `businessNote` 例：汽车街「二手车 B2B 拍卖平台…」、大搜车「据公开报道整体 GMV 量级约数百亿…待 IPO 招股书披露」

## §3 Tab ② 行业横向对比 — 详细字段清单

渲染入口 `renderCompare()`。

### 3.1 顶部控件（ctrl）

- **矩阵对比期**下拉：23Q1..26Q1 任选一季度
- 注释：「下表与分析按所选季度计算；人人车/大搜车未上市无数据，不纳入对比。」（原型文字，含「人人车」字样 —— 疑为淘车车旧名，**待澄清**）

### 3.2 全维度横向对比矩阵（card）

- **列**：瓜子★ + 6 家上市（汽车之家/CarMax/Carvana/理想/优信/易鑫）+ 行业均值（蓝）+ 瓜子 vs 行业（delta）
- **行分组（4 组）**：

| 分组 | 指标（key / label / kind） |
|------|---------------------------|
| 业务规模与效率 | GMV(亿元,amt) / 交易量(万台,amt) / 变现率(%,pct) / 单车收入(元/台,amt) |
| 成长性 | 营收同比(%,growth) / GMV 同比(%,growth) |
| 盈利与财务健康 | 营收 / 毛利率(%) / 净利润 / 净利率(%) / 经营现金流 / 现金 / 有息负债 |
| 估值水平（仅上市可比） | 市值 / PE(TTM,x) / PS(TTM,x) / EV/Sales(x) |

- 瓜子列黄底加粗；负值红字；行业均值蓝字加粗；delta 按 kind 输出 pp/x/% 并着色
- 瓜子未上市的估值列显示「未上市」灰字

### 3.3 分维度横向分析（takeaways，四段）

- 一、业务规模与效率：GMV 排名（含瓜子位次）、变现率瓜子 vs 行业（含 principal/marketplace 口径不可比提示）、单车收入对比
- 二、成长性：营收同比排名、GMV 同比瓜子 vs 行业
- 三、盈利与财务健康：盈利分化（已盈利/仍亏损名单）、净利率差距、现金 vs 有息负债覆盖
- 四、估值水平：PS/PE 区间与高低、瓜子未上市需同业倍数反推、peers 横跨二手车平台/美股零售/NEV 整车可比性有限

### 3.4 ★ 对瓜子的启示与行动建议（insight，蓝框，6 点）

含 **IPO 估值锚反推**：上市同业 PS 均值 × 瓜子 TTM 营收 → 对应市值，再扣流动性折让 15–30% 给区间。
其余：规模→变现/利润、变现率杠杆测算、盈利对标路径、资金面与投资人结构启示、对外叙事差异化。
注释：「估值反推为示例测算，实际需结合增速、盈利质量与市场窗口调整。」

> [!warning] 合规提示
> 此区块含**对未上市主体的估值测算与 IR 叙事建议**，涉及投资判断。交付前需确认是否在 IR 内部使用范围内、是否需法务/合规审阅。

### 3.5 单指标深度对比（deep）

- **控件**：指标下拉（按 业务/财务/估值 分组，11 指标）+ 区间下拉（起→止季度）+ 公司 chips（可勾选）
- **三张卡片（grid）**：
  1. 横向对比柱状图（含「行业均值」虚线参考线）
  2. 时序趋势折线图（多公司 + 行业均值虚线）
  3. 估值-增速散点图（X=营收增速 YoY%，Y=PS(TTM)，仅上市）

## §7 待澄清问题（按优先级）

> 以下为原型中无法单方面判断、需向客户确认的点。

### P0 — 决定工作范围

1. **竞品最终数量与构成**：原型 10 家（瓜子 + 6 上市 + 3 未上市）是定稿，还是示例？原有 17 家美股经销商是否完全弃用？理想（NEV 整车）是否真纳入二手车对标组？
2. **原定 13F 交叉持股报告交付物是否废止**？新模版里 13F 退化为单竞品前 50 大持有人 —— 确认 `cross_holding.py` 矩阵报告不再需要？
3. **看板技术栈**：继续 Streamlit（重构成双 Tab），还是按 HTML 原型纯前端重写（原型是原生 HTML+SVG，无 Plotly/Streamlit）？

### P1 — 决定数据源与合规

4. **媒体/公众号数据采集方式**：后端自动抓取，还是分析师人工录入策展？tier 3 公众号/社媒的版权与合规边界？
5. **卖方研报评级/目标价/分布数据源**：公开新闻聚合（原型注释「非完整 consensus」），还是采购 Wind/Choice 一致预期？
6. **HKEXnews 覆盖范围**：就易鑫/理想/汽车之家 3 家，还是更广？港股持有人是否需 CCASS 数据？
7. **业务指标（GMV/交易量/变现率）来源**：未上市竞品（瓜子）数据谁录入、口径如何统一？已上市美股是否披露 GMV？
8. **行情数据**：免费延迟行情可接受，还是需付费实时？

### P2 — 决定模型与口径

9. **8 类投资人性质分类规则**：13F 申报人不直接给 8 类，映射规则由谁定义？港股持有人分类口径？
10. **「对瓜子的启示」中的 IPO 估值锚反推**：是否在交付范围内（涉及对未上市主体的估值建议，需法务/合规审阅）？
11. **币种与口径统一**：美元/港币/RMB 混合，折算规则（汇率来源、季末 vs 均值）？原型各竞品 `fx` 字段已标口径，需统一规则。
12. **数据时滞要求**：tier 1 官方 T+1？tier 2/3 日级？季报季是否需日内？
13. **原型文字「人人车/大搜车未上市」**：原型注释提到「人人车」，但 COMPANIES 列表里是「淘车车」—— 是否为同一主体旧名，还是遗漏的竞品？
